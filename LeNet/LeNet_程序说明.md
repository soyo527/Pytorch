# LeNet 项目小白说明

这份项目是在做一件很具体的事：

用一个经典卷积神经网络 `LeNet`，去识别 `FashionMNIST` 数据集里的衣物图片属于哪一类。

`FashionMNIST` 可以理解为“服饰版手写数字集”。  
每张图片都是 `28 x 28` 的灰度图，一共分成 10 类，比如 T 恤、裤子、鞋子、包等。

---

## 1. 这个项目里每个文件是干什么的

### `model.py`

定义神经网络结构，也就是“模型长什么样”。

你可以把它理解成：

- 输入一张图片
- 先经过卷积层提取局部特征
- 再经过池化层缩小尺寸
- 然后拉平成一维向量
- 最后经过全连接层输出 10 个类别分数

### `model_train.py`

负责训练模型。

主要做 4 件事：

1. 读取数据
2. 把训练集切分成训练集和验证集
3. 让模型一轮一轮学习
4. 保存验证集效果最好的模型参数

### `model_test.py`

负责测试模型。

主要做 2 件事：

1. 加载训练好的模型参数
2. 用测试集检验准确率，并打印部分预测结果

### `plot.py`

不负责训练，也不负责测试。  
它只是把数据集里的图片画出来，帮助你直观理解“模型输入的原始数据到底是什么样子”。

---

## 2. 程序整体流程

整个程序可以按下面这条线去理解：

1. `plot.py` 先让你看看数据长什么样
2. `model.py` 定义模型结构
3. `model_train.py` 读取训练数据并训练模型
4. 训练过程中挑出“验证集表现最好”的参数，保存到 `best_model.pth`
5. `model_test.py` 加载 `best_model.pth`，在测试集上验证效果

可以把它想象成：

- `model.py` 是“搭建大脑”
- `model_train.py` 是“让大脑学习”
- `model_test.py` 是“考试验收”
- `plot.py` 是“先看看教材长什么样”

---

## 3. 数据是怎么流动的

下面按“图片进入模型以后经历了什么”来讲。

### 第一步：读取图片

在 `FashionMNIST` 里，每张图片的原始大小是：

`1 x 28 x 28`

这 3 个数字分别表示：

- `1`：通道数，灰度图只有 1 个通道
- `28`：图片高度
- `28`：图片宽度

如果一次送进去 32 张图片，那么张量形状就会变成：

`32 x 1 x 28 x 28`

其中最前面的 `32` 叫做 `batch_size`，表示一批有多少张图。

---

## 4. LeNet 模型内部做了什么

### 输入

输入张量形状：

`[batch_size, 1, 28, 28]`

---

### 第 1 层：卷积层 `c1`

```python
self.c1 = nn.Conv2d(in_channels=1, out_channels=6, kernel_size=5, padding=2)
```

它的作用是：

- 用 6 个卷积核去扫描图片
- 提取一些基础特征，比如边缘、纹理、简单轮廓

因为设置了 `padding=2`，所以卷积后高宽不变：

`[batch_size, 1, 28, 28] -> [batch_size, 6, 28, 28]`

也就是说：

- 图片还是 28 x 28
- 但通道从 1 变成了 6
- 说明模型提取出了 6 组不同特征

---

### 第 2 层：激活函数 `Sigmoid`

```python
self.sig = nn.Sigmoid()
```

激活函数的核心作用是：

给网络加入非线性能力。

如果没有激活函数，很多层叠起来也还是类似线性变换，模型表达能力会很弱。

---

### 第 3 层：平均池化 `s2`

```python
self.s2 = nn.AvgPool2d(kernel_size=2, stride=2)
```

池化层的作用是：

- 缩小特征图尺寸
- 减少参数量和计算量
- 保留比较重要的整体信息

这里会把尺寸减半：

`[batch_size, 6, 28, 28] -> [batch_size, 6, 14, 14]`

---

### 第 4 层：卷积层 `c3`

```python
self.c3 = nn.Conv2d(in_channels=6, out_channels=16, kernel_size=5)
```

这一步继续提取更复杂的特征。

因为这里没有设置 padding，所以高宽会缩小：

`[batch_size, 6, 14, 14] -> [batch_size, 16, 10, 10]`

---

### 第 5 层：平均池化 `s4`

```python
self.s4 = nn.AvgPool2d(kernel_size=2, stride=2)
```

再次减小尺寸：

`[batch_size, 16, 10, 10] -> [batch_size, 16, 5, 5]`

---

### 第 6 步：拉平 `flatten`

卷积层输出的是三维特征图，但全连接层需要的是一维向量，所以要先拉平：

`[batch_size, 16, 5, 5] -> [batch_size, 400]`

因为：

`16 x 5 x 5 = 400`

---

### 第 7 步：全连接层

```python
self.f5 = nn.Linear(400, 120)
self.f6 = nn.Linear(120, 84)
self.f7 = nn.Linear(84, 10)
```

这几层可以理解为“最后的分类判断部分”。

形状变化是：

- `[batch_size, 400] -> [batch_size, 120]`
- `[batch_size, 120] -> [batch_size, 84]`
- `[batch_size, 84] -> [batch_size, 10]`

最后输出 10 个数，每个数对应一个类别的打分。

注意：

这里输出的是“分数”而不是“概率”。  
在训练时，`CrossEntropyLoss` 会自动处理这些分数，所以最后一层不用手动写 `Softmax`。

---

## 5. `forward()` 到底是什么

`forward()` 可以理解成：

“规定一张图片进入模型之后，要按什么路线往前走。”

也就是：

1. 先过卷积层
2. 再过激活函数
3. 再过池化层
4. 重复一次卷积和池化
5. 拉平
6. 过全连接层
7. 输出分类结果

当你写：

```python
output = model(batch_x)
```

实际上底层就会自动调用：

```python
output = model.forward(batch_x)
```

## 6. `model_train.py` 训练阶段到底在干什么

训练代码最核心的是这几个步骤。

### 6.1 读取并划分数据

项目会先读取 `FashionMNIST` 的训练集，然后分成两部分：

- 80% 作为训练集
- 20% 作为验证集

为什么不能只用训练集？

因为如果你只看训练集准确率，模型可能只是“背答案”，并不是真的学会了。  
所以还需要验证集来检查模型对新数据的表现。

---

### 6.2 DataLoader 的作用

`DataLoader` 的作用是：

- 按批次读取数据
- 自动打乱训练数据
- 把图片和标签一批一批送进模型

比如设置：

```python
batch_size=32
```

就表示每次拿 32 张图片来训练，而不是 1 张 1 张来。

这样做的好处是：

- 训练更快
- 梯度更稳定

---

### 6.3 优化器 `Adam`

```python
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)
```

优化器的职责是：

根据梯度去调整模型参数，让损失越来越小。

你可以把它想成：

模型犯错以后，优化器负责“怎么改”。

---

### 6.4 损失函数 `CrossEntropyLoss`

```python
criterion = nn.CrossEntropyLoss()
```

损失函数的作用是：

衡量“模型预测结果”和“真实答案”之间差得有多远。

- 差得越大，损失越大
- 差得越小，损失越小

训练的目标，本质上就是让损失越来越小。

---

### 6.5 一个训练 batch 的完整流程

每拿到一批数据，都会做下面几步：

#### 第 1 步：前向传播

```python
output = model(batch_x)
```

让图片经过网络，得到预测分数。

#### 第 2 步：计算损失

```python
loss = criterion(output, batch_y)
```

比较预测和真实标签之间的差距。

#### 第 3 步：清空旧梯度

```python
optimizer.zero_grad()
```

PyTorch 默认会把梯度累加，所以每轮都要先清零。

#### 第 4 步：反向传播

```python
loss.backward()
```

根据损失，计算每个参数应该往哪个方向改、改多少。

#### 第 5 步：更新参数

```python
optimizer.step()
```

真正把参数改掉，让模型朝更好的方向前进。

---

## 7. 什么是 epoch

`epoch` 指的是：

模型把整个训练集完整学一遍。

比如训练集有 48000 张图片，`batch_size=32`，那一个 epoch 会分很多个 batch 才跑完。

如果 `num_epochs=20`，就表示把整个训练集来回学 20 遍。

---

## 8. 为什么训练时还要验证

每个 epoch 结束后，代码都会在验证集上跑一遍。

目的不是更新参数，而是观察：

当前这版模型，对没参与训练的数据表现怎么样。

这能帮助我们判断：

- 模型是不是在真正变强
- 有没有过拟合

项目里还做了一件很重要的事：

只要发现“这一轮验证准确率更高”，就把这一轮的模型参数记下来。

训练结束后，会把模型恢复成“验证集上最好”的那一版，并保存为：

`best_model.pth`

这叫做“保存最佳模型”。

---

## 9. `model_test.py` 测试阶段做了什么

测试阶段的流程是：

1. 创建同样结构的 `LeNet`
2. 把 `best_model.pth` 加载进来
3. 读取测试集
4. 用测试集计算准确率
5. 额外打印几张测试图片的预测类别和真实类别

这里的测试集和训练集、验证集不同：

- 训练集：用来学习
- 验证集：训练过程中观察效果
- 测试集：训练全部结束后，做最终检验

---

## 10. `plot.py` 为什么把图片放大到 224

这个脚本里的：

```python
transforms.Resize(size=224)
```

只是为了展示更清楚。  
因为 `28 x 28` 的原图太小，直接看会比较模糊。

这不代表模型训练时要用 `224 x 224`。  
训练代码里模型依然使用 `28 x 28` 输入。

## 11. 训练输出里常见字段是什么意思

### `Train Loss`

模型在训练集上的平均损失。  
一般来说越小越好。

### `Train Acc`

模型在训练集上的准确率。  
表示训练集里有多少比例被分对了。

### `Val Loss`

模型在验证集上的平均损失。

### `Val Acc`

模型在验证集上的准确率。  
这个指标通常很重要，因为它更能反映模型的泛化能力。

## 14. 这次我顺手帮你修正了什么

为了让代码逻辑和注释真正对应上，我顺手修了两处明显问题：

1. 原来训练集和验证集的 DataLoader 指向了同一份划分后的数据，这样会导致“训练”和“验证”实际上看的是同一批样本，不利于理解，也不利于正确评估模型。
2. 原来保存模型参数的写法不正确，训练结束时可能会报错。现在已经改成了标准的 `torch.save(model.state_dict(), 路径)`。
